RP2W Help

Setting up robot computer (server)

  1. Open the "Settings" window by selecting the item "Change Settings..." in the "Tools" menu.
  2. Select the "Communication" tab in the "Settings" window.
  3. If in the "Run As" group box is selected option "Server (Robot)", continue with step 4. Otherwise select the option "Server (Robot)", click on the "Update and close" button in the bottom right corner of the "Settings" window and go back to the step 1.
  4. Fill other boxes:
    • Tx Timer Interval, Pan Delay and Tilt Delay are values related to communication between server application and the robot. The robot may not work properly with other than default values.
    • Robot stop timeout: When server doesn't receive response from the remote computer for longer than this time, it stops robot's motors.
    • Program stop timeout: When server doesn't receive response from the remote computer for longer than this time, it closes connection with remote computer, disables remote controlling, restarts Wireless Network Adapters and re-enables remote controlling.
    • Port (required): Number of the first port which application uses for network communication. The application uses seven consecutive port numbers. You should have this in mind when setting up firewalls or routers. The valid range for this option is 1024 to 65529.
    • IP Address, Login and Password are not used for server settings.
    • Enable Remote Controlling after startup: If checked, server will automatically enable remote controlling mode after the application starts.
    • Video Device (required): Select video device that you want to use for videoconferencing.
    • Audio Device (optional): Select audio device that you want to use for videoconferencing.
    • Data COM Port (required): Select COM port to which is connected robot.
  5. Click on the "Update and close" button in the bottom right corner of the "Settings" window.

Note: Due to "restart wireless network adapters" functionality, the robot application has to run under account with administrator rights.

Setting up the remote computer (client)

  1. Open the "Settings" window by selecting the item "Change Settings..." in the "Tools" menu.
  2. Select the "Communication" tab in the "Settings" window.
  3. In the "Run As" group box select option "Client (Remote)".
  4. Fill other boxes:
    • Tx Timer Interval, Pan Delay, Tilt Delay, Robot stop timeout and Program stop timeout are values related to server application. Changing these on the remote computer won't take any effect.
    • Port (required): Number of the first port which application uses for network communication. The application uses seven consecutive port numbers. You should have this in mind when setting up firewalls or routers. This value has to be the same as on the robot computer.
    • IP Address (required): IP address of the robot computer. If the robot computer has more IP addresses, it has to be the one, which is visible from the remote computer (public IP). (It has to be numerical IP address. It can't be hostname.)
    • Login and Password are not used for server settings.
    • Video Device (optional): Select video device that you want to use for videoconferencing.
    • Audio Device (optional): Select audio device that you want to use for videoconferencing.
    • Data COM Port is not used for client settings.
  5. Click on the "Update and close" button in the bottom right corner of the "Settings" window.

Establishing connection

  1. On the robot computer, click on the item "Enable Remote Controlling" in the "Server" menu (if it is not already enabled).
  2. On the remote computer, click on the button "Connect to Remote Device" in the "Client" menu.

When grayed out items on the main form of client application become active and status bar refreshes network connection latency, battery status and wifi signal strength info, the connection has been established. Otherwise see Troubleshooting paragraph.

Note 1: Enabling remote controlling on the robot computer disables posibility of controlling the robot from this computer. The robot can be controlled only from remote computer that is connected to the robot computer.

Note 2: When client (remote) is connected to the server (robot), it is possible change some of the server's settings from the client.

Videoconferencing functionality

On the remote computer, there are available some videoconferencing features after connecting to the server. In the menu "Client" are buttons:

  • Enable Local WebCam Stream: Enables streaming from remote computer. If local WebCam stream is not enabled, it will not be possible to watch client's video feed on the robot computer.
  • Switch Robot Computer To VideoConf Mode: Remotely shows a maximized window with client's video feed on the robot computer.
  • Close VideoConf Mode On Robot Computer: Remotely hides the window with client's video feed on the robot computer.

There is also button that looks like microphone. It mutes/unmutes local microphone.

In the menu "Tools" are available functions that are shared between server and client applications.

  • Show Local Camera Window...: Shows a window with video feed from selected local video device. (On the remote computer will show a window with black background, if the local webcam stream is not enabled.)
  • Show Remote Camera Window...: Shows a window with video feed from server (on client) or client (on server). It will show window with black background on the server, if the client's webcam stream is not enabled.

Troubleshooting

During network communication several problems can occur. Some of them can be solved by restarting the applications. If you need to restart the application on the robot computer, you can try to use "Restart Robot Application" function in the "Client" menu on remote computer. It can be used even when remote is not connected to the robot computer, but the robot computer has to have remote controlling enabled. If server application is restarted remotely by "Restart Robot Application" function, it will re-enable remote controlling automatically after application restarts. Restarting client and server usually solves problems with video stream (e.g. freezed video) and problems with reconnecting after disconnecting.

What if it is not possible to establish connection?

  • Check out if both remote and robot computer are connected to the right network.
  • Check if it is possible to reach robot computer's IP address from remote computer. (You can use ping command in the Windows Command Prompt)
  • Try to disable firewalls on the robot and the remote computers. (If this step solves the problem, you should re-enable firewalls and add exception for the RP2W application to them. Remember, RP2W is using seven consecutive ports in total. The one you have set in the settings is the first of this sequence.)

It may be helpful to use debug window (Tools->Show debug window...) when you are solving problems. You can see whether applications are sending something on network or not.

When you'll need our assistance with solving problems, please send us detailed log files. They will be created by applications after enabling extended debug logging (Tools->Extended Debug Logging Enabled). Logs will show up on remote's or robot's desktop named as DEBUG_CLIENT_LOG.TXT or DEBUG_SERVER_LOG.TXT after you close remote or robot application. Please enable extended logging right after you start the applications (before you establish connection).

Note: Both debug window and extended debug logging are timeconsuming features. Using them may cause slower response time of the application.
